
## 接口概览

本接口用于查询大模型资源池中所有 Worker 节点的实时状态与资源使用情况。

可根据具体情况进行数据聚合处理，以满足业务需求。

---

## 一、接口定义

- **请求方式**：`GET`

- **请求路径**：`/v1/workers`

- **协议与端口**：内网请求 HTTP 

生产环境地址（健康云环境可访问）：192.168.120.44:9998

**鉴权 token**：gpustack_4f50a5c7bad4fd01_7c054bcb8d3cf921e9b1fa43517fa81b

### 1.1 地址示例

```http
GET http://211.90.240.240:30001/v1/workers?page=1&perPage=10
```

此为测试环境地址

---

## 二、请求头

|参数|必填|类型|示例|说明|
|---|---|---|---|---|
|Authorization|是|string|Bearer gpustack_xxx|鉴权令牌，格式：`Bearer {token}`|
|Accept|否|string|application/json|指定响应内容类型，推荐固定使用 `application/json`|

---

## 三、请求参数（Query）

|参数|必填|类型|默认|范围|示例|说明|
|---|---|---|---|---|---|---|
|page|否|int|1|≥1|1|当前页码，从 1 开始|
|perPage|否|int|10|1–100|10|每页记录数，上限 100|

> **示例**：
> 
- `?page=2&perPage=20`：查询第 2 页，每页 20 条数据。
- 不传参则默认 `page=1, perPage=10`。

---

## 四、响应头

|参数|说明|
|---|---|
|HTTP/1.1 200 OK|请求成功|
|Content-Type|application/json; charset=utf-8|
|Content-Length|响应体字节长度|
|Cache-Control|no-cache|
|Date|响应生成的 UTC 时间（RFC 7231 格式）|
|Server|服务端框架或服务器名称（示例：uvicorn）|

---

## 五、响应体

### 5.1 顶层结构

|字段|类型|说明|
|---|---|---|
|items|array|本页 Worker 列表|
|pagination|object|分页信息|

#### pagination 对象

|字段|类型|说明|
|---|---|---|
|page|int|当前页码|
|perPage|int|每页记录数|
|total|int|总记录数|
|totalPage|int|总页数|

### 5.2 items 元素（Worker 详情）

| 字段              | 类型          | 说明                                        |
| --------------- | ----------- | ----------------------------------------- |
| id              | int         | Worker 全局唯一标识                             |
| name            | string      | 用户定义的节点名称                                 |
| hostname        | string      | 节点主机名                                     |
| ip              | string      | 节点 IP 地址                                  |
| port            | int         | 节点对外 RPC 端口号                              |
| labels          | object      | 自定义标签，键值对形式；常见键：`os`、`arch`、`worker-name` |
| system_reserved | object      | 系统预留资源（ram：预留内存（字节），vram：预留显存（字节））        |
| state           | string      | 节点状态，枚举值：`ready` / `not_ready`            |
| state_message   | string/null | 状态描述，例如心跳丢失时的错误提示                         |
| **status**      | object      | 资源使用详情（详见下节 Status 结构）                    |
| unreachable     | boolean     | 网络可达性，`false` 表示可连通；`true` 表示网络隔离或无回应     |
| heartbeat_time  | string      | 最近心跳时间，UTC ISO 8601 格式                    |
| created_at      | string      | 记录创建时间，UTC ISO 8601 格式                    |
| updated_at      | string      | 记录最近更新时间，UTC ISO 8601 格式                  |

#### status 对象结构

##### 1. cpu (object)

- **total** (int)：CPU 核心总数
    
- **utilization_rate** (float)：整体利用率，单位 % （如 0.4463 表示 0.4463%）

##### 2. memory (object)

- **total** (long)：总内存字节数
    
- **used** (long)：已用内存字节数
    
- **allocated** (long)：已分配内存字节数
    
- **utilization_rate** (float)：利用率，单位 %
    
- **is_unified_memory** (boolean)：是否为统一内存架构

##### 3. swap (object)

- **total** (long)：交换分区总字节数
    
- **used** (long)：已用交换字节数
    
- **utilization_rate** (float)：利用率，单位 %

##### 4. filesystem (array)

每项对象包含：

- **mount_point** (string)：挂载点路径
    
- **mount_from** (string)：设备或源
    
- **total**/**used**/**free**/**available** (long)：对应字节数

##### 5. os (object)

- **name** (string)：操作系统名称（如 Ubuntu）
    
- **version** (string)：版本号（如 22.04.5 LTS）

##### 6. kernel (object)

- **name** (string)：内核名称（如 Linux）
    
- **release** (string)：发行版本（如 4.19.90-...）
    
- **version** (string)：内核版本号与编译信息
    
- **architecture** (string)：架构（如 aarch64）

##### 7. uptime (object)

- **uptime** (float)：自上次启动以来运行时长（秒）
    
- **boot_time** (string)：最近一次启动时间，UTC ISO 8601 格式 

##### 8. gpu_devices (array)

每项对象包含：

- **index** (int)：设备编号
    
- **name**/**vendor**/**type** (string)：设备名称/厂商/类型（cuda/npu）
    
- **core** (object)：
    
    - **total** (int)：核心总数
        
    - **utilization_rate** (float)：核心利用率（%）

- **memory** (object)：
    
    - **total**/**used**/**allocated** (long)：显存总/已用/已分配字节数
        
    - **utilization_rate** (float)：显存利用率（%）
        
- **temperature** (float)：设备当前温度（℃）
    
- **labels** (object)：设备自定义标签

##### 9. rpc_servers (object)

该字段 **不** 使用表格展示，而是以键值对形式描述：

- **键**：字符串形式的设备 `index`（与 gpu_devices 中的 `index` 一致）
    
- **值**：进程信息对象，包含：
    
    - **pid** (int)：RPC 服务进程 ID
        
    - **port** (int)：该进程监听的 RPC 端口
        
    - **gpu_index** (int)：对应的设备编号
